# Compel — Бизнес-результаты (ROI)

> Источник: интервью 2026-03-27

## Оптимизация импорта (8ч → 40мин)
- Основной бизнес-эффект: **ускорение цикла разработки**, а не частоты обновления каталога
- Разработчик тратил на выкатку фичи, связанной с импортом, 40 минут вместо целого рабочего дня
- Данных о влиянии на актуальность каталога/цен для клиентов нет

## Latency 3500 → 95ms
- **Проактивная оптимизация** — сделана до появления конечных пользователей
- Прод существовал, но им ещё никто не пользовался
- Визуально разница заметна на всех стендах (dev и другие)
- Данных о влиянии на конверсию нет (пользователей не было)

## Kafka pipeline (замена батчевого импорта)
- Kafka pipeline **полностью заменил батчевый импорт** — 40-минутный импорт перестал быть нужен
- Каждый компонент обновляется поштучно в реальном времени
- Визуально для пользователей незаметно, но система стала проще, нагрузка снизилась

## BDD — снижение багов
- **-70% багов в Jira** по бэкенд-разработке (подтверждено статистикой, были специальные дашборды)
- После внедрения BDD на фронтенде — **примерно такое же снижение** относительно их базового уровня

## Microservice template
- **~10 сервисов запущено** через шаблон (план был 15-20)
- Впереди ещё **~50 сервисов** — масштаб значительно вырос

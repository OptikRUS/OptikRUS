# Fortech — CI/CD & Quality Improvement

## Case: Восстановление инженерных практик

- Problem (S/T):
  - На проекте отсутствовали линтеры, тесты, type checkers
  - Инструменты были в зависимостях, но заигнорены
  - Code review тратилось на мелкие замечания по форматированию

- Action (A):
  - После дейли предложил команде: "Если я восстановлю все линтеры и type checkers в одном MR, будете ли все поддерживать?"
  - Получил buy-in от всей команды
  - Восстановил линтеры, type checkers в одном MR
  - Написал тесты, довёл coverage до 92%

- Result (XYZ):
  - Сократилось время code review — автоматизация ловит форматирование
  - Выработался единый code style
  - Команда перестала тратить время на code review стилистики
  - Повысилось доверие к коду за счёт тестов
  - Каждый мог изолированно делать задачи на проекте
  - Ускорение CI/CD цикла ~40%

## Engineering Culture Impact
- Buy-in через демо, а не через приказ
- Один MR как "proof of concept" для всей команды
